## Summary

Bulk seed endpoint.

---

## Description

**As a** QA engineer  
**I want** an admin endpoint that seeds up to 50,000 randomised alerts in one call  
**So that** I can load the database with realistic volume for performance and pagination tests without running the seed script in a loop

---

## Acceptance criteria

### AC1 — Bulk seed endpoint with count limit

- [ ] `POST /api/v1/dev/seed-bulk?count=N` (admin JWT, non-production only) inserts `N` randomised alerts; max `count` is 50,000; values above the limit return `422`
- [ ] Alerts are inserted with varied severities, statuses, and sources so filter and pagination tests have a realistic distribution
