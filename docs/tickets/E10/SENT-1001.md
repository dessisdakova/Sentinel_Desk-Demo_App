## Summary

Test reset endpoint.

---

## Description

**As a** QA engineer  
**I want** an admin-only API endpoint that truncates all app data tables and re-runs the seed script  
**So that** automated tests can reset the database to a known baseline between runs without restarting Docker or using the CLI manually

---

## Acceptance criteria

### AC1 — Reset endpoint with environment guard

- [ ] `POST /api/v1/test/reset` is available only when `ENVIRONMENT != production`; returns `403` with `code=FORBIDDEN_IN_PRODUCTION` if called in production

### AC2 — Full truncate and re-seed

- [ ] Calling the endpoint truncates all application data tables (alerts, cases, playbook_runs, audit_logs, etc.) and re-runs `backend/scripts/seed.py` logic — uses the shared seed module, not a duplicate
- [ ] Returns `200` with `{ "status": "ok", "seeded_entities": {...} }` on success

---

## Technical notes

- Truncate app tables only; re-run existing `backend/scripts/seed.py` (shared seed module — TEST_DATA.md §3.1)
